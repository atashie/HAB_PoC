## 1. Domain

- **Space:** State of Florida. Boundary = US Census 2022 cartographic boundary (states, 1:20M),
  reprojected to **EPSG:5070** (CyAN's grid). FL polygon area ≈ **154,175 km²**. Source (cited):
  `https://www2.census.gov/geo/tiger/GENZ2022/shp/cb_2022_us_state_20m.zip`, accessed 2026-07-02
  (cached under `models/acquire/_cache/`, gitignored). A finer boundary (500k) is a later refinement.
- **Time:** **OLCI only, 2016-04-24 → present** (MERIS 2008–2012 excluded, D-21). The local archive
  holds **531 weekly composites**, latest `L20261722026178` (2026 day 172–178, ~late June 2026).
- **Target grid:** native CyAN **300 m** weekly max-composite, EPSG:5070.

## 2. CyAN coverage in Florida (measured, not assumed)

Measured from the already-local OLCI CONUS mosaics, clipped to the FL polygon (16 evenly-spaced weeks
across the record; union for the footprint):

| Quantity | Value | Note |
|----------|-------|------|
| Pixels inside FL polygon | 1,713,049 | all FL land + water in the analysis window |
| **Water-pixel footprint** (ever-valid, DN ≤ 253) | **≈ 128,628 px** | **≈ 11,577 km²** |
| Mean valid water / week | ≈ 116,783 px | rest of the footprint is cloud-masked (DN 255) that week |
| Mean cyano **detection** (DN 1–253) / week | ≈ 23,990 px | most valid water is DN 0 (below-detection) |

**Sanity check (passes):** the ~11,577 km² footprint closely matches Florida's real ≈12,000 km² of
surface water — confirming the boundary clip isolates **inland/within-boundary freshwater**, not the
Gulf/Atlantic. (An un-clipped lat/lon box reads ≈1.46M "valid" px ≈131,000 km², i.e. it sweeps in
shallow marine water CyAN also processes — which is *not* our freshwater-HAB target and is why the
clip is mandatory.)

**Modeling implication — class balance:** only ~20% of *observed* water pixels show any detection
(DN ≥ 1) in a week, and bloom-threshold exceedance will be a smaller fraction still. This is the
~8–14%-minority reality that drives the imbalance handling and event-based metrics in `DESIGN.md` §8.

## 2b. Lake universe / Florida mask (the operative unit)

Per D-23, the modeling unit is the **resolvable lake**. The Florida lake set = the **CyAN
resolvable-lakes shapefile** (COMID-keyed) clipped to the FL polygon — the same lake list the EPA
forecast serves (its panel carries `State`, so `State = FL` gives the benchmark's FL lakes directly).
Per-lake CyAN features = **mean/median/SD of CI over each lake's resolvable pixels** (Schaeffer recipe;
`DESIGN.md` §2). Exact FL lake count is pulled in the acquisition step below (order of a few hundred).
COMID enables **identity joins** to WQP/NARS/NWIS/LakeCat — the reason for the lake-level pivot.

## 3. Compute gate — CONTEXT (pixel measurement; superseded by the lake pivot)

Fused-table sizing (one row per pixel-week; multi-horizon adds target *columns*, not rows):

| | Value |
|---|---|
| Water footprint × OLCI weeks | 128,628 × 531 = **≈ 68.3 M rows** |
| Dense float32 × ~100 feature cols (monolith) | **≈ 27.3 GB** → **UNSAFE** on a 16–32 GB box (no headroom; the sampled union *undercounts* the true footprint, so the real figure is higher) |
| One week's rows, partitioned | ≈ 0.05 GB → trivially fine |

**~~Decision (D-22): native pixel~~ — SUPERSEDED by D-23 (lake-level).** The pixel table was *feasible*
(≈68.3 M rows, viable via partitioned columnar storage), so compute never forced the choice. The
**EPA-matching pivot** moved us to lake-level for **comparability + COMID identity joins**, not
compute. At lake-level, **N ≈ FL lakes × 531 weeks ≈ tens of thousands of rows** — trivially in RAM,
and **all four models run at full scale (no subsampling)**. This section stays as evidence that the
pixel-vs-lake choice was made on scientific grounds, not a compute limitation.

## 4. Per-source fresh-pull plan (Florida, OLCI POR)

All pulls scripted + cached + manifested (sha256), consistent with the `data-sources/` discipline.
**Join key = the lake (COMID)**; each lake carries its resolvable-pixel footprint for CyAN aggregation.

| Source | What to pull for FL | Method / code | Join to lake |
|--------|--------------------|----------------|--------------|
| **CyAN resolvable-lakes** (the mask/universe) | shapefile of resolvable-lake footprints (COMID), clipped to FL | from `data-sources/cyan/` / CyAN project (NHDPlus `NHDWaterbody`) | defines the lakes |
| **CyAN CI** (target + feature) | **Already local** — per-lake weekly mean/median/SD over each lake's resolvable pixels (OLCI) | `data-sources/cyan/` archive; aggregate in `models/` | per-lake stats |
| **ERA5** (weather) | 2 m temp, total precip, surface solar, 10 m wind (u/v), evaporation, MSL pressure; weekly + antecedent; FL, POR | `data-sources/weather/access/era5_cds.py` (CDS API) | overlying 0.25° cell(s) |
| **NWIS** (hydrology) | FL gages: discharge + gage height, daily; state FIPS 12 | `dataRetrieval` (planned in `data-sources/NWIS/`) | COMID/NLDI/HUC/nearest + staleness |
| **WQP** (water quality) | FL stations: TP/TN/NH₃-N/chl-a/water-temp/turbidity/EC/DO/pH/Secchi; **WQX2/3 reconciliation** | `data-sources/WQP/` (planned access code) | COMID/NLDI/HUC/nearest + staleness |
| **BasinATLAS L12 + morphology** (static) | FL sub-basin attributes + lake mean depth & surface area (EPA's morphology predictors) | `D:/BasinATLAS_Data_v10` (`bchars_intersected12.csv` / `.gdb`); morphology from NHD/LakeCat | containing L12 sub-basin + lake |

**Order of work:** resolve the FL **resolvable-lakes mask** first (defines the lake universe), then
per-lake CyAN aggregation + AL1 target (labels), then static morphology/BasinATLAS join (cheap), then
ERA5 (gridded), then the sparse in-situ networks (NWIS, WQP — COMID/NLDI joins + staleness are the
hard part).
